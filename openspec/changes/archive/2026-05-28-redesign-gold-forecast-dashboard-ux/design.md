## Context

当前 GoldFXGraph dashboard 已经能展示最新 XAUUSD 研究结果、结构化交易字段、K 线图、多个 agent 摘要和风险提示，但整体布局偏“模块堆叠”，首屏重点不够集中，信息密度与视觉层级在桌面端和移动端都还有优化空间。

这次需求明确限定为纯展示层重设计：不允许修改任何后端接口、请求路径、字段名、数据契约、路由、权限、query key 或业务逻辑。设计目标是在保持所有信息完整可见的前提下，让页面更像一个专业的金融研究 cockpit，而不是普通后台详情页。

## Goals / Non-Goals

**Goals:**
- 重新组织 dashboard 的视觉层级，让核心结论、当前价格、方向、置信度和结构化交易字段在首屏最先被扫到。
- 提升信息密度与可读性平衡，减少“同权重模块并列”的感觉。
- 统一卡片体系、状态标签、边距、圆角、阴影、渐变和字体层级。
- 改善 chart、列表、风险区和状态区的协同关系，让它们在视觉上分层明确。
- 让 loading、empty、error 和 success 的视觉反馈更专业、清晰且不误导。
- 让移动端、平板端和桌面端都能保持同一套信息优先级。

**Non-Goals:**
- 不修改任何 API 接口、请求参数、返回字段解析和数据模型。
- 不修改任何 TypeScript 类型字段名称。
- 不修改 store、query key、权限逻辑、路由逻辑。
- 不新增 mock data、不创造新的业务字段、不改变 dashboard 既有功能。
- 不引入新的复杂交互、交易逻辑、自动化动作或后台能力。

## Decisions

### 1. 采用“首屏决策区 + 次级证据区 + 深度分析区”的三层信息架构
当前页面中很多信息都同等重要，导致首屏缺少明确焦点。重设计后将页面拆成三层：
- 首屏决策区：当前价格、方向、置信度、最新执行时间、核心结论、关键交易字段
- 次级证据区：K 线图、市场快照、summary card、agent 投票与风险提示
- 深度分析区：历史表现、透明度信息、免责声明

这样能让用户先看到结论，再看证据，再看辅助信息。  
备选方案是保持现在的模块堆叠再做视觉增强，但会继续放大“信息平铺”的问题，因此不采用。

### 2. 使用“深色研究面板 + 控制色金色/青色”的 restrained premium 风格
页面需要高级感，但不应走过度霓虹或 cyberpunk 路线。视觉基调采用深色金融研究面板，辅以少量金色强调与青色数据高亮：
- 背景保持深色渐变与低噪点网格
- 核心价格和方向用高对比强调
- 置信度、关键结论和重点字段使用受控亮色
- 卡片保持统一边框与柔和阴影，避免强烈发光效果

备选方案是更强烈的 neon/cyberpunk 风格，但这会削弱研究工具的可信度，并可能与高密度数据展示冲突。

### 3. 将信息卡片统一成少量标准组件，减少视觉碎片化
当前页面存在多种卡片样式并行的问题。重设计后统一为几类基础视觉单元：
- hero summary card
- metric card
- evidence card
- risk card
- chart card
- table/card hybrid

这样能保证不同区域的视觉语言一致，并让读者通过样式快速判断信息类型。  
备选方案是为每个区域做完全独立的视觉风格，但会增加认知负担，降低页面整体性。

### 4. 让 chart 与摘要、风险区形成“主次关系”，而不是并列抢焦点
K 线图是核心证据之一，但它不应压过首屏结论。设计上将 chart 保持更强的展示面积与更少的干扰信息，同时通过右侧或下方的指标区补充上下文。  
备选方案是把 chart 进一步扩大为绝对主视觉，但会压缩结构化交易字段和多 agent 结论的可见性，不利于研究场景。

### 5. 响应式采用“桌面分栏、平板两列、移动单列”的稳定折叠策略
桌面端采用 12 栅格分区，确保核心信息优先；平板端压缩为两列主结构；移动端改为单列纵向堆叠，并对次级信息做合并与折叠式处理。  
备选方案是移动端也保持大量并列卡片，但会导致阅读疲劳和首屏过长。

## Risks / Trade-offs

- [Risk] 首屏重排后，用户可能不适应信息顺序变化 → [Mitigation] 保持字段名称与内容不变，仅调整视觉权重，并让核心字段仍在首屏显眼位置。
- [Risk] 深色高对比风格容易“过亮”或过重 → [Mitigation] 控制发光面积、降低阴影强度、只让少数重点元素使用亮色。
- [Risk] 响应式重构可能引入布局塌陷 → [Mitigation] 优先保持现有组件数据流不变，只通过结构和 class 调整实现断点适配，并在 375/768/1024/1440 进行检查。
- [Risk] 卡片统一化后可能显得过于模板化 → [Mitigation] 用标题层级、重点标签和局部强调区分不同信息类型，而不是完全同质化。
