## Why

GoldFXGraph 目前虽然已经能生成 XAUUSD 研究结果并在前端展示，但交互和结果表达仍然偏“单次查询 + 手动刷新”的模式，和研究系统应有的持续自动运行体验不一致。用户希望取消手动刷新，把所有定时任务统一收敛到一个 15 分钟周期内自动执行，并且在页面上直接看到最新执行时间与多智能体执行状态。

与此同时，当前 forecast 的方向表达仍然更接近单点结论。用户确认后，希望把结果改成“时间窗口方向区间”的形式，由 agent 给出今天之后不同时间段的看多 / 看空 / 震荡判断，而不是单独做一个未来 3 天预测。这更符合研究型输出，也更适合在首页首屏直接扫描当前价格、方向、入场、止盈和止损。

## What Changes

- 取消前端手动刷新入口，以及后端用于手工触发研究运行的接口。
- 将原本分散的自动任务统一为一个 15 分钟周期的统一调度流程，按固定顺序执行：
  - 获取最新黄金数据
  - 更新当日最新黄金数据
  - 执行多智能体黄金日线分析
  - 持久化 research run、forecast 和执行状态
- 新增或扩展后端运行状态结构，让前端可以读取：
  - 最新执行时间
  - 当前是否正在执行
  - 当前执行到哪个阶段
  - 每个 agent 的执行状态
- 将 forecast 的方向表达从单点判断扩展为“固定时间窗口”的区间判断，例如：
  - 0-3 天
  - 3-5 天
  - 6-15 天
  - 15 天后
  并允许 agent 额外补充一个自定义观察窗口说明。
- 首页首屏重排，优先展示当前价格、市场方向、区间方向、入场点位、止盈、止损和最新执行时间。
- 保持多 agent 分析原因列表、K 线图、最新市场新闻和情绪模块不变。

## Capabilities

### New Capabilities

- `scheduled-research-run-loop`: 统一以固定 15 分钟周期执行黄金数据获取、日线更新、agent 分析和结果持久化。
- `windowed-forecast-direction`: 让 agent 对固定时间窗口输出方向区间，而不是只返回一个单点方向。
- `live-run-status-dashboard`: 前端能够读取并展示最近一次执行时间、运行中状态和各 agent 状态。

### Modified Capabilities

- `backend-research-api`: 删除手工触发研究的路径，增加统一调度的状态查询能力，并返回窗口化 forecast 结构。
- `langgraph-forecast-workflow`: 保持现有 agent 分工，但最终输出改为多时间窗口方向区间。
- `gold-forecast-dashboard`: 首屏优先展示价格、方向区间和交易建议，同时展示统一调度状态。
- `forecast-persistence`: 增强持久化模型，以记录统一调度的运行状态和窗口化 forecast 结果。

## Impact

- 后端 `src/goldfxgraph/` 的 API、workflow、scheduler、persistence 和 schema 模块都会受影响。
- 前端 `apps/web/` 需要移除手动刷新按钮，新增调度状态展示，并重排首页首屏信息。
- 数据库需要支持统一调度运行状态与窗口化 forecast 的持久化。
- 现有的技术分析、宏观分析、新闻分析、K 线图和市场新闻/情绪内容应继续保留，不需要重写。
