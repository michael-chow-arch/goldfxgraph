## 背景

GoldFXGraph 目前已经具备完整的研究型 Dashboard 页面，可以展示最新 XAUUSD 预测结果、结构化交易字段、多智能体投票、OHLC 与免责声明。现有页面功能完整，但用户希望进一步强化视觉辨识度，并尝试一种更有冲击力的主题：**黑绿金赛博科技风**。

这次讨论的重点不是新增功能，而是重新组织现有内容的视觉层级和信息顺序，让页面更像一个“专业研究终端”，而不是普通的数据面板。

## 目标

- 建立一套适合 XAUUSD 研究场景的黑绿金赛博视觉方案。
- 让页面第一眼就能传达“这是黄金研究终端”，而不是通用 Dashboard。
- 调整内容展示顺序，使信息流更符合研究决策路径。
- 保持现有功能不变，不引入自动交易、不新增复杂数据源逻辑。

## 非目标

- 不改动后端研究 API 的语义。
- 不新增自动轮询、自动交易或券商接入。
- 不把页面做成娱乐化、游戏化 HUD。
- 不引入新的数据结构，只重排现有信息。

## 视觉方向

### 1. 整体气质

采用“**黑底 + 霓虹绿主导 + 金色点睛 + 少量冷青辅助**”的组合。

- 黑底负责建立高对比和专业感。
- 绿色负责表达数据活跃、信号流动和技术感。
- 金色负责强调黄金资产身份和关键结论。
- 冷青只作为结构线、边缘光、次级状态使用，避免抢主视觉。

推荐气质关键词：

- 赛博
- 研究终端
- 机构级
- 高对比
- 玻璃拟态
- 金属边缘光
- 轻微扫描感

### 2. 颜色建议

建议色板如下：

- 背景主色：`#020805`
- 背景层次：`#04110A`、`#07150E`
- 主绿：`#2EE66B`、`#19C37D`
- 金色：`#F5C542`、`#D4AF37`
- 辅青：`#27C7FF`
- 主文本：`#EAF7EE`
- 次级文本：`#8FA8A0`
- 卡片表面：`rgba(4, 17, 10, 0.72)` 到 `rgba(255, 255, 255, 0.06)`

使用原则：

- 绿色用于当前信号、方向、置信度、进度条和状态高亮。
- 金色只用于价格、关键结论、黄金资产身份与重点提醒。
- 黑底必须是深墨绿黑，不要纯黑死板。
- 透明玻璃卡要有实底，不要过度透明到影响可读性。

### 3. 字体建议

- 中文正文：`Noto Sans SC`
- 中文标题：`Noto Serif SC` 或加粗版 `Noto Sans SC`
- 数字、时间、编号、价格：`Fira Code`

目标是形成“技术感 + 金融感 + 中文可读性”的组合，而不是纯装饰风。

### 4. 视觉效果建议

- 玻璃拟态卡片，但透明度要克制。
- 关键卡片加轻微绿光或金色边缘光。
- 背景可加极淡网格线或扫描线，增加终端感。
- 价格、方向、置信度可使用柔和发光，不要强烈霓虹外扩。
- hover 效果以边框、光泽和阴影变化为主，不做大幅缩放。

## 核心内容与顺序

### 第一屏：现在是什么信号

第一屏只回答一个问题：**当前黄金研究结果指向什么？**

优先展示：

- 当前价格
- 方向
- 置信度
- 数据时间
- 数据来源
- 运行状态

推荐展示方式：

- 左侧：大号当前价格 + 方向徽章 + 置信度条
- 右侧：数据时间、来源、运行编号、预测编号
- Hero 只保留最关键的 2 到 3 个 chips，避免重复堆叠

### 第二层：为什么是这个信号

这里放分析摘要，回答“为什么会得到这个方向”：

- 技术分析摘要
- 宏观分析摘要
- 新闻分析摘要
- 风险分析摘要

展示方式建议：

- 2x2 或 4 卡横向结构
- 每张卡只保留 2 到 4 行核心结论
- 长文本默认截断或折叠，避免一屏被文字占满

### 第三层：多智能体是否一致

这一层是页面的“研究可信度层”，建议作为重点模块突出。

显示内容：

- 技术分析 Agent 投票
- 宏观分析 Agent 投票
- 新闻分析 Agent 投票
- 风险分析 Agent 投票
- 预测规划 Agent 投票

展示方式建议：

- 桌面端用表格，突出结构化比较
- 移动端用卡片列表，避免横向拥挤
- 方向徽章使用绿色、金色、冷青等一致的状态编码
- 理由默认只展示首句或摘要，支持展开阅读

### 第四层：该怎么做研究决策

这里展示的是“可执行研究字段”，但不是自动交易信号。

显示内容：

- 入场价
- 止盈价
- 止损价
- 风险回报比
- 建议持有周期
- 日内建议
- 中长期建议

展示方式建议：

- 做成独立“行动卡”
- 入场 / 止盈 / 止损采用并列卡片
- 风险回报比以小徽章或角标突出
- 日内建议和中长期建议分成两个独立块，减少混淆

### 第五层：支撑信息

最后展示研究透明度信息，不抢主结论。

显示内容：

- OHLC
- 预测编号
- 免责声明

这些内容属于“支撑与解释层”，应该在视觉上弱于前面四层。

## 页面布局建议

推荐采用以下结构：

1. 顶部 Hero
2. 研究摘要与风险提示
3. 智能体投票矩阵
4. 交易研究字段
5. 元数据与 OHLC
6. 免责声明

布局重点：

- 桌面端可使用 12 栏网格。
- 首屏保持较强对比和明确焦点。
- 下方模块不要重复展示同一信息。
- 重要内容只出现一次，避免“信息重复感”。

## 组件建议

### 1. Hero 价格卡

- 放大当前价格。
- 用绿色显示方向和置信度。
- 金色只用于黄金资产身份和关键状态。

### 2. 摘要卡

- 每个摘要卡承载一个分析维度。
- 内容尽量短、准、结构化。

### 3. 投票矩阵

- 以表格为主，便于横向比较不同 Agent。
- 卡片模式用于移动端适配。

### 4. 行动卡

- 把入场、止盈、止损、周期、日内建议和中长期建议组合成一组。
- 强化“研究决策支持”而非“执行交易”。

### 5. 元数据卡

- 提供数据来源、时间、运行编号和预测编号。
- 作为研究透明度说明。

## 风险点

- 绿色过多会像游戏界面，不像研究终端。
- 金色过多会显得俗气，失去专业感。
- 黑底如果没有层次，会显得沉闷。
- 透明玻璃如果过轻，会造成可读性问题。
- 投票表格如果信息过密，移动端会难以阅读。
- 如果多个区域重复展示方向、价格和置信度，页面会显得冗余。

## 验收标准

- 页面第一眼能明显看出黑绿金赛博科技风。
- 首屏能清楚看到当前价格、方向、置信度、时间与来源。
- 内容顺序符合“信号 → 理由 → 共识 → 行动 → 支撑信息”。
- 多智能体投票成为核心模块，而不是附属信息。
- 页面中不会重复堆叠同一组核心数据。
- 所有文案保持中文。
- 不引入自动交易、自动轮询或新的数据源逻辑。

## 结论

这次设计的重点不是“把页面做得更花”，而是让现有信息被更清楚地组织起来。黑绿金赛博风适合 GoldFXGraph 的黄金研究主题，只要控制好颜色占比和信息层级，它既能一眼吸引人，也能保持专业研究终端的可信度。
